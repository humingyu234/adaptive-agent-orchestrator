# ProjectContext - 项目文件上下文模块

## 1. 问题

Agent 执行时如何获取项目文件结构和关键文件信息？

痛点：
- Agent 不知道项目有哪些文件
- 无法快速定位关键配置或代码
- 缺乏统一的文件上下文抽象

它在整个系统里的位置：
- 属于 `project-context` CLI 命令
- 可被 Agent 通过 `reads` 声明引用

---

## 2. 决策

实现 `ProjectContext` 类，提供项目文件扫描和摘要能力。

核心对象：
- `ProjectContext` - 上下文管理器
- `ProjectStructure` - 项目结构摘要
- `FileInfo` - 文件元信息
- `FileSummary` - 文件内容摘要

主要能力：
- `scan()` - 扫描项目结构
- `get_file_summary()` - 获取单个文件摘要
- `find_files()` - 按模式查找文件
- `get_context_for_agent()` - 为 Agent 提取上下文

排除规则：
- 默认排除 `__pycache__`, `.git`, `.venv`, `node_modules` 等
- 默认排除 `.env`, `*.pyc` 等敏感或编译文件

---

## 3. 取舍

### 备选方案

1. **实时读取文件系统**
   - 优点：信息实时
   - 缺点：每次都要遍历，性能差
   - 未采用

2. **缓存 + 按需刷新（采用）**
   - 优点：性能好，支持强制刷新
   - 缺点：需要管理缓存失效
   - 采用原因：项目结构变化不频繁，缓存合理

### 牺牲

- 不做文件内容变更监听
- 不做跨项目关联

### 换来

- 快速的项目结构扫描
- 统一的文件上下文接口
- 可被 Agent 声明式引用

---

## 4. 当前边界

**未覆盖：**
- 文件内容变更监听
- Git 历史集成
- 跨项目引用

**保持简单：**
- 只扫描本地文件系统
- 只提供基础文件摘要

## 6. 面试 / 博客表达

### 一句话版本

ProjectContext 提供项目文件结构扫描和摘要能力，让 Agent 能感知项目上下文。

### 稍展开版本

ProjectContext 是一个项目文件上下文管理器，通过扫描项目结构、提取文件摘要、支持模式查找，为 Agent 提供统一的文件上下文访问接口。它支持缓存、排除规则和语法检查，帮助 Agent 快速理解项目结构。
