# LiveInterruptController - 实时中断控制器

## 1. 问题

运行时如何支持人工干预？如何在不重启的情况下暂停、恢复或中止任务？

痛点：
- 运行时无法干预正在执行的任务
- 无法动态修改查询或注入上下文
- 缺乏统一的运行时控制接口

它在整个系统里的位置：
- 属于 Scheduler 运行时控制层
- 支持外部信号注入
- 与 StateCenter 配合实现状态快照

---

## 2. 决策

实现 `LiveInterruptController` 类，提供运行时中断控制能力。

核心对象：
- `LiveInterruptController` - 中断控制器
- `InterruptSignal` - 中断信号枚举
- `InterruptRequest` - 中断请求
- `InterruptResponse` - 中断响应
- `InjectCommand` - 注入命令枚举

支持的中断信号：
- `PAUSE` - 暂停执行
- `RESUME` - 恢复执行
- `ABORT` - 中止任务
- `INJECT` - 注入命令
- `SKIP` - 跳过当前 Agent

支持的注入命令：
- `MODIFY_QUERY` - 修改查询
- `ADD_CONTEXT` - 添加上下文
- `FORCE_COMPLETE` - 强制完成
- `OVERRIDE_RESULT` - 覆盖结果
- `CHANGE_TARGET` - 更改目标

---

## 3. 取舍

### 备选方案

1. **信号量 + 全局状态**
   - 优点：实现简单
   - 缺点：线程安全问题，难以扩展
   - 未采用

2. **事件驱动 + 队列（采用）**
   - 优点：线程安全，支持多种信号
   - 缺点：实现稍复杂
   - 采用原因：符合运行时控制需求

### 牺牲

- 不支持分布式中断
- 不支持中断回滚

### 换来

- 线程安全的中断控制
- 多种中断信号支持
- 可扩展的注入命令

---

## 4. 当前边界

**未覆盖：**
- 分布式运行时中断
- 中断操作回滚
- 中断持久化恢复

**保持简单：**
- 只支持单进程内中断
- 中断历史只保存在内存

---

## 5. 后续演化

- 支持中断持久化
- 集成 Web UI 控制面板
- 支持分布式中断信号

### 稍展开版本

LiveInterruptController 是一个实时中断控制器，通过事件驱动和队列机制，支持运行时暂停、恢复、中止任务，以及动态注入修改查询、添加上下文等命令。它为人工干预运行时提供了统一的控制接口。
